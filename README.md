# LSK_TGC_Analizer
Две модели, одна выдает топ 3 категории канала, другая полное его описание, написан файлик для удобного использования и внедрения моделей в проекты. Модель с топ3 категории была дообучена на данных(за основу взята RuBERT), которые я самостоятельно собрал и предобработал. Вторая модель - интегрированная LLM.

Дообучение RuBert(файл texttoclass):
Система автоматической классификации Telegram-каналов по тематикам на основе анализа последних публикаций.
Технологии: Hugging Face Transformers, PyTorch, Telethon (Telegram API) для асинхронного сбора данных, Scikit-learn (Label Encoding), регулярные выражения для чистки текста
Модель анализирует не один пост, а совокупность текстов, выдавая топ-3 доминирующих категории канала. Реализован удобный пайплайн предсказания, от ввода юзернейма канала до получения финальной тематики.

Весь проект:
Система автоматического анализа Telegram-каналов, объединяющая классификацию тематик на основе RuBERT и суммаризацию контента через LLM-интеграцию (OpenRouter).

## Основные возможности
* Автоматический парсинг: Асинхронный сбор сообщений через `Telethon`.
* NLP-классификация: Определение тематики канала (IT, Блоги, Финансы и др.) с помощью дообученной нейросети.
* LLM-генерация: Создание summary(краткого описания) о деятельности канала на основе проанализированных постов.
* Гибкость: Поддержка анализа последних N постов с фильтрацией шума.

## Технологический стек
`PyTorch`, `Transformers`, `Telethon`, `OpenRouter API`, `NLTK`, `PyMorphy2`, `Pandas`

## Структура проекта
```text
├── sessions/               # Папка для файлов сессии Telegram (.session)
├── preprocess/
│   └── clean.py            # Логика очистки и лемматизации текста
├── inference/
│   ├── category.py         # Классификатор на базе RuBERT
│   └── summary.py          # Генератор описания канала через LLM
├── models/
│   └── category_model/     # Сюда необходимо поместить веса модели
├── parser/
│   └── telegram.py         # Модуль парсинга данных
├── config.py               # Конфигурация (API ключи и пути)
├── main.py                 # Точка входа в приложение
└── requirements.txt        # Зависимости
```

## Установка и настройка
Подготовка модели:
* Скачайте веса модели `RuBERT`.(вот ссылка на них https://drive.google.com/drive/folders/1q4yaWOa_gPgVwI5zSiHcRLaeCk9mc7C_?usp=sharing)
* Поместите их в директорию `models/category_model/`

Конфигурация:
Создайте файл `config.py` (или скопируйте из примера) и замените на свои

Telegram Session:
При первом запуске скрипт запросит номер телефона для авторизации в Telegram. Файл сессии будет автоматически создан в папке `sessions/`. Если у вас уже есть файл сессии, просто поместите его в эту папку.

## Запуск
```bash
python main.py
```
Введите юзернейм канала (например, `@MAIuniversity`) в консоли, и система выдаст отчет о тематиках и краткое описание канала.
